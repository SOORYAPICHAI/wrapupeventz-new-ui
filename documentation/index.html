Project Structure

The structure of the Berserk HTML template:

  • css
    • assets - all common styles used on the pages
    • components - styles for specific elements
    • skins - color styles
  • img - images for the project
  • js
    • assets - all common scripts used on the pages
    • components - scripts for specific elements
  • vendor - additional libraries used on our project

HTML

The default page template, where you can replace the Page Title, Favicon, and put your CSS, JavaScript and the page content:

<!DOCTYPE html>
<html class="brk-preloader" lang="en" data-brk-skin="brk-blue.css">
<head>
  <title></title>
  <meta charset="utf-8">
  <meta name="viewport" content="width=device-width,height=device-height,initial-scale=1,maximum-scale=1">
  <meta name="format-detection" content="telephone=no">
  <link rel="shortcut icon" href="favicon.ico">
  <link rel="apple-touch-icon-precomposed" href="favicon.ico">
  <meta name="theme-color" content="#2775FF">
  <meta name="keywords" content="themeforest, theme, html, template">
  <meta name="description" content="themeforest, theme, html, template">
  <link rel="stylesheet" id="brk-direction-bootstrap" href="css/assets/bootstrap.css">
  <link rel="stylesheet" href="https://use.fontawesome.com/releases/v5.0.13/css/all.css" integrity="sha384-DNOHZ68U8hZfKXOrtjWvjxusGo9WQnrNx2sqG0tfsghAvtVlRW3tvkXWZh58N9jp" crossorigin="anonymous">
  <link id="brk-base-color" rel="stylesheet" href="css/skins/brk-base-color.css">
  <link rel="stylesheet" href="css/assets/offsets.css">
  <link id="brk-css-min" rel="stylesheet" href="css/assets/styles.min.css">

  [-- CSS -- ]

</head>
<body>
  <div class = "builder-page">
  </div>

  [-- CONTENT -- ]

  [-- JAVASCRIPT -- ]

  <script src="js/scripts.min.js"></script>
</body>
</html>

In our template we have used Bootstrap 4, here you can read more about this library: https://getbootstrap.com/docs/4.1/layout/overview/

Async Loader

All our components are using their own set of CSS and JavaScript files, which grouped to libraries. Lets check how it works on the Alerts example. If you check the first line of this element, then you will notice data-brk-library attribute:

<div data-brk-library="component__alert" class="alert alert-clean fade alert-primary alert-dismissible text-left font__family-montserrat font__size-16 font__weight-light show" role="alert">

This is mean that then a page will be loaded, our script will search for all elements with this attribute, after this try to search for such library in our berserk.js file (or in the scripts.js file if you are using original compressed version) and will add all required CSS and JS files. List of used files you can find on the documentation for each element.

If for some reason you wish to change the source folder path then you need to override the Berserk.settings.project_prefix variable and type your project path. This variable also can be found on the berserk.js file (or in the scripts.js). For example this is how it could looks: 

Berserk.settings.project_prefix = '/your/custom/project/path/';

You can avoid this method and add all CSS and JS files manually, but for this you also need to remove the data-brk-library attribute on the element. If you wish to deeper inspect this async method, then please open the js\assets\unminimized\berserk.js file and find the line var libraries = which contains all libraries and their dependencies.

Installation

After you download your website template from ThemeForest and make all of your edits you're going to need to upload the files to your web server. You will need an FTP client such as TotalCommander or FileZilla. In your FTP client add your server details (get this from your web host provider) and then you can connect to your server and upload your files. After this your website should work correctly.

How to start?

You can edit project in 2 ways:

  • Manual HTML editing - This method is most flexible, because gives you the 100% control at the project, but you need to be familiar with HTML, CSS and JS. In this case you don't need novi folder, and you can upload to your server all folders except this one. After this you can create a new empty page and add our elements from the pages which you like and edit it, or you can use our original pages and just edit them - this is preferred way, because we worked hard to make the pages looks great and well balanced. In this method you can leave only the pages which you like and remove the not used.
  • Page Builder using - You can use the builder which is placed in the novi folder. It allow you to easily add any elements on the page and edit the images, text, links and other attributes. Unfortunately the builder doesn't allow you to edit any HTML attributes, like some sliders or dynamic blocks, but you can use buil-it HTML, CSS and JavaScript editor to behave this. Please read the documentation about this editor, so you will be able to use it on full power. Once you completed the project configuration, then you can download the archive with project and upload it to the server, or you can edit it directly on the server and just configure the Publish path, so the project will be saved to your server home directory. This is described more detailed in the Page Builder Documentaion.

Updates

You have lifetime free updates of this template, which can be downloaded from the themerest.com -> Downloads section. Don't forgot to make backup of your live website before applying any updates. You need to replace all folders and files which placed on the template archive, especially make sure that /css /js /vendor folders are updated. Sometimes it will be required to update also the HTML structure, so if some of your edited element is stopped working then compare it structure with structure at our live website: http://html.berserk.nikadevs.com/.

Customizations

We strongly recommend to add your own CSS and JS code customizations as separated files and add them to the project via LINK and SCRIPT tag. The suggested css file name is /css/custom.css. The suggested JS filename is /js/custom.js. In this case you will be able to easily update the folder under the /css and /js folder there is our scripts and styles a stored. Don't forgot to add your CSS file at the end of styles list or to add !important suffix, so your styles will override our own. In our JS files we are used behaviors - this is helpful functions which can be easily overriden. Just copy the entire required block of the code, didn't change the Behaviors object function name and update the function body code as you need. In this case your code will override our function and will be used instead.

Skin Color

In raw html version we used the CSS Variables for color controls. This is mean that you can easily change the website colors in one place and you can even test how it will looks before, right in the browser development console.

All predefined skin files placed in the /css/skins folder, so if you wish to change your skin to another then you can change path to this file in your project, or connect required file after all other CSS files, so it will override previous settings.

For example this file after all CSS files will change color skin to the Green skin: 

<link rel="stylesheet" href="css/skins/brk-green.css">

The Page Builder currently doesn't support CSS Variables, because they are still waiting for approvements from W3C. So we used the previous stable method for this: css variables were replaced with their real values and all css files stored in their own folders, which you can find on the novi\projects\template\css\ folder. So if you wish to change the skin file, then you should edit the folder name on the line 16

<link id="brk-base-color" rel="stylesheet" href="css/brk-blue/skins/brk-base-color.css">

for example to this: 

<link id="brk-base-color" rel="stylesheet" href="css/brk-green/skins/brk-base-color.css">

RTL

To make your website in RTL mode you need to make next steps:

  • Add dir = "rtl" attribute to the html tag
  • Replace the css\assets\bootstrap.css file with css\assets\bootstrap-rtl.css
  • Replace the css\assets\offsets.css file with css\assets\offsets-rtl.css

If you are not familiar with the RTL mode, then this usually mean that it doesn't need you. More information about this you can read here : https://en.wikipedia.org/wiki/Right-to-left

Retina

To make your image retina ready you should use data-srcset attribute instead of data-src, so instead of this: 

<img data-src="image.jpg" class="lazyload">

you need to use this type of code:

<img data-srcset="responsive-image1.jpg 1x, responsive-image2.jpg 2x" class="lazyload">

If you are using a background image, then instead of data-bg attribute you should use data-bgset, so instead of this: 

<div class="lazyload" data-bg="img/1170x562_2.jpg"></div>

you need to use this type of code:

<div class="lazyload" data-bgset="img/1170x562_1.jpg 1x, img/2240x1104_1.jpg 2x"></div>

This structure is used because our website implements the lazyload script, which is reduce the amount of requests to your server and loads images only then users scroll to them. This also speed up the website loading time.

Google Maps

To use the Maps Embed API, you must register your app project on the Google Cloud Platform Console and get a Google API key which you can add to your app or website. To get your key, please check this page: https://developers.google.com/maps/documentation/embed/get-api-key

Once you configured your Google Maps account, then you need to replace the minimized js/scripts.min.js with js/scripts.js. After this open this file and search for the Berserk.settings.gmap_api_key variable. Usually it's placed near the line 40. Type in this variable your google maps api key, so it will looks like: 

Berserk.settings.gmap_api_key = "BYzaSyCSzRKxiq0GL_aWvND_lYwciIe4moNzNMY";

Recaptcha

To use the Google ReCaptcha, you must register your app project on the Google reCAPTCHA service page. The reCAPTCHA v3 returns a score for each request without user friction. The score is based on interactions with your site and enables you to take fine-grained action. Register reCAPTCHA v3 keys here.

Once you configured your Google reCAPTCHA v3 account, then you need to replace the minimized js/scripts.min.js with js/scripts.js. After this open this file and search for the Berserk.settings.recaptcha_api_key variable. Usually it's placed near the line 41. Type in this variable your google maps api key, so it will looks like: 

Berserk.settings.recaptcha_api_key = "REzaSyCSzRKxiq0GL_aWvND_lYwciIe4moNzNMY";

After this open the php/subscribe.php file and type your reCAPTCHA secret key in the $secretKey variable which is usually placed on the line 9, so it will looks like:

$secretKey = "9Faru5QQBGHTTExjV7mfo_dEDo-12nEne7D7IchX";
